# Onboarding

This is the short entry point for understanding the repository. It routes to canonical sources rather than replacing them. If this file conflicts with a canonical source, trust the canonical source and fix this file.

## First Reads

Use this order for fresh orientation:

1. `README.md` for purpose, structure, and public-facing context.
2. `AGENTS.md` for hot agent routing.
3. `docs/attention-management.md` for how actions are enforced.
4. `docs/directory-layout.md` before file placement changes.
5. `docs/documentation-style.md` before Markdown, KB, or skill edits.
6. `skills/planning/SKILL.md` before broad multi-step work.
7. `skills/` for other reusable workflows.

## Opening Protocol

For orientation, status, or "what next" requests, answer in this order:

1. one sentence saying this is a repository for skills, working knowledge, and reflective notes;
2. one sentence naming the relevant active area;
3. one sentence naming the highest-priority next work or uncertainty;
4. links to the most relevant source files.

## Current Shape

The repository is organized as a retrieval and action system:

- `AGENTS.md` is hot routing for future agents.
- `README.md` is the public entry point.
- `docs/` contains shared process and attention-management rules.
- `skills/` contains reusable workflows.
- `.codex-plugin/` and `.agents/plugins/` make this repository installable as a Codex plugin.
- `projects/`, `concepts/`, and `observations/` contain durable knowledge.

## Sync Rule

Keep this file short. Update it only when the repository purpose, first-read order, major directory structure, local skill map, or attention-management system changes.
